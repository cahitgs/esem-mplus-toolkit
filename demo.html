the workflow, step by step

01

The data

data.dat is space-delimited, no header, dot decimals — Mplus free format. In the app, Data auto-detects the 25 columns; if your own file is a semicolon CSV or uses comma decimals, the app exports a clean Mplus-ready .dat for you.

Download data.dat Open the Data step
02

Build the model

On the Λ target-pattern matrix set four factors (M1–M6 → F1, M7–M12 → F2, M13–M18 → F3, M19–M24 → F4), pick a rotation (Geomin or Target), the estimator (MLR), and which models to generate. A live syntax preview and a conceptual diagram update as you click.

▶ Open the model step (4 factors preloaded)
03

Generate the syntax

The app writes exact Mplus .inp for each model — CFA, ESEM (Geomin/Target), bifactor-ESEM, and the invariance sequence — wrapped under the 90-character limit. ESEM outputs include SVALUES, which you'll need for ESEM-within-CFA. Download one file or a .zip of all.

▶ Open the syntax step
04

Run it in your Mplus

This is the only step outside the app. Open each .inp in your own licensed Mplus and save the .out. For this demo the runs are already done — every .out in example-dataset/ is a real Mplus 8.3 result that converged with full standard errors and fit.

05

Read it as APA

Drop the .out files back in. Each becomes a fit row, a standardized loadings table (target loadings bold, δ uniqueness, ω, factor correlations), auto-written prose, and an interactive path diagram. The six invariance models become one comparison table with Satorra–Bentler scaled Δχ², ΔCFI, ΔRMSEA and verdicts. Export with Copy for Word or Download .docx.

▶ Open: CFA + ESEM + bifactor ▶ Open: invariance sequence ▶ Open: bifactor diagram
06

Convert to ESEM-within-CFA

Because the ESEM .out carries a SVALUES block, Results shows an ESEM-within-CFA card: it picks a referent indicator per factor, fixes that referent's cross-loadings to their ESEM values, and writes a runnable CFA you can embed in larger SEM / MIMIC / DIF models (Morin's referent method). Verified in Mplus to reproduce the source ESEM exactly.
