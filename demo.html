the workflow, step by step

01

The data

data.dat is space-delimited, no header, dot decimals — Mplus free format (16 columns: items i1–i15 + gender). When you load your own file, the app auto-detects columns and can export a clean Mplus-ready .dat from a semicolon CSV or comma decimals.

Download data.dat Open the Data step
02

Build the model

On the Λ target-pattern matrix set three factors (i1–i5 → F1, i6–i10 → F2, i11–i15 → F3), pick a rotation (Geomin or Target), the estimator (MLR), and which models to generate. A live syntax preview and a conceptual diagram update as you click.

▶ Open the model step (3 factors preloaded)
03

Generate the syntax

The app writes exact Mplus .inp for every model — CFA, ESEM (Geomin & Target), bifactor-CFA, bifactor-ESEM, and the six invariance steps — wrapped under the 90-character limit. ESEM outputs include SVALUES for ESEM-within-CFA. Download one file or a .zip of all — or skip the downloads entirely: auto mode (Chrome/Edge) writes everything into a local working folder with a RUN-ALL.bat; double-click it once and the results import themselves as each model finishes. An optional Convergence aids panel adds Morin's remedies for stubborn models: a loosened CONVERGENCE criterion and per-item residual > 0 constraints for Heywood cases.

▶ Open the syntax step
04

Run it in your Mplus

This is the only step outside the app. Open each .inp in your own licensed Mplus and save the .out. For this demo the runs are already done — all eleven .out files in example-dataset/ are real Mplus 8.3 results that converged with full standard errors and fit.

05

Read it as APA

Drop the .out files back in. Each becomes a fit row, a Morin-style loadings table (Hoyle Handbook, Table 2): items grouped into subscale blocks with a per-block ω row, target loadings bold, non-significant loadings italic — plus auto-written prose and an interactive path diagram. Drop several together and they share one fit-comparison table. Copy for Word and Download .docx export the very same tables, in the same layout.

▶ All five measurement models ▶ CFA + ESEM + bifactor ▶ Bifactor-CFA
06

Bifactor diagram

The diagrams follow the article's figures and are interactive — toggle cross-loadings / uniquenesses, show/hide factors, drag the ellipses to reposition, and export SVG / PNG. The bifactor layout puts the three specific factors on one side and the global factor on the other, reaching every item.

▶ Open the bifactor-ESEM result
07

Invariance across groups

Drop the six inv_*.out files together (men vs women) and they collapse into one comparison table: configural → metric → scalar → strict → variance–covariance → latent means, with Satorra–Bentler scaled Δχ², ΔCFI, ΔRMSEA, and a "supported / not supported" verdict (Chen, 2007).

▶ Open the invariance sequence
08

Invariance across time

The same six steps also run longitudinally — one single-group model per step, with each wave as its own ESEM set, loadings equated across waves, and every indicator's residual correlated with itself over time. Sequences are generated for CFA, ESEM, and bifactor-ESEM (Morin's Hoyle-Handbook chapter). This demo drops twelve .out files from a 2-wave well-being example: measurement invariance holds through variance–covariance, and the latent-means step correctly flags the built-in growth — the bifactor sequence even shows it lives on the general factor only.

▶ Open the longitudinal sequences
09

Convert to ESEM-within-CFA

Because the ESEM .out carries a SVALUES block, Results shows an ESEM-within-CFA card: it picks a referent indicator per factor, fixes that referent's cross-loadings to their ESEM values, and writes a runnable CFA you can embed in larger SEM / MIMIC / DIF models (Morin's referent method). Verified in Mplus to reproduce the source ESEM exactly (here: df 63, log-likelihood −11107.13 in both).
